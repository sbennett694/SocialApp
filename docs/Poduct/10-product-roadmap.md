ProSocial Product Roadmap

This roadmap translates the ProSocial philosophy into development phases
that gradually move the product from prototype → public community platform.


================================
Phase 1 — Core Infrastructure
Goal: Make the platform stable and usable
================================

1. Notification Reliability
- Ensure notifications always open the correct content
- Scroll and highlight referenced items
- Improve navigation between feed items and content

2. Club Membership UX
- Join / request / member button states
- Confirmation feedback
- Leave club functionality

3. Member Permissions
Allow moderators to configure member abilities:

members can:
- post highlights
- comment
- volunteer
- create suggestions

4. Seed Data System
Improve the seed scenario engine:

support:
--default
--all 3

populate:
- join requests
- highlights
- projects
- milestones
- comments

5. Automated Tests
Add tests for:

- club membership logic
- notification navigation
- project routing
- feed rendering


================================
Phase 2 — Community Foundation
Goal: Introduce meaningful social interaction
================================

6. Club Commons Feed
Each club has an activity feed showing:

- highlights
- suggestions
- comments
- milestones
- volunteer actions

7. Project Activity Timeline
Projects maintain a history of activity:

- tasks completed
- milestone updates
- volunteer activity
- comments

8. Club-Centered Projects
Allow moderators to create projects from the club page.

Projects belong to a club and act as collaborative goals.

9. Feed Card Types
Clearly identify activity types:

New Comment
New Suggestion
New Question
New Gratitude
New Highlight

Include icons and color indicators.

10. Notification Preview
Replace internal IDs with preview text:

show:
first 5 words of activity


================================
Phase 3 — Collaboration Tools
Goal: Enable deeper collaboration between users
================================

11. Volunteer System
Members can volunteer to help with:

- projects
- milestones
- tasks

Moderators receive notifications and can assign help.

12. Gratitude System
Users can publicly thank others for contributions.

Example:
"Thanks to Sarah for helping with the UI redesign!"

13. Mentions
Support:

@username

Mentioned users receive notifications.

14. Role System
Expand club roles:

member
helper
organizer
moderator
owner


================================
Phase 4 — Personal Social Layer
Goal: Strengthen relationships between users
================================

15. Private Friends List
Users maintain private contacts with:

- notes
- reminders
- personal events

16. Profile Privacy Fields
Profile sections with visibility control:

Private Life
- interests
- goals
- notes
- biography

17. Skill Tags
Users can add skill tags:

coding
design
photography
carpentry

Projects can request specific skills.


================================
Phase 5 — Engagement & Community Growth
Goal: Encourage participation and community momentum
================================

18. Achievements
Feed events celebrating progress:

John completed milestone: Launch Website

19. Community Challenges
Clubs can create shared challenges:

monthly goals
project sprints
community competitions

20. Opportunity Feed
Surface opportunities to help:

Projects needing volunteers
Clubs looking for members
New collaboration opportunities


================================
Long-Term Vision
================================

ProSocial becomes a platform where communities form around
shared effort and meaningful collaboration.

Users don't just share content.

They:

- build projects
- help each other
- learn skills
- support community progress

The platform becomes a coordination layer that helps
people organize and accomplish things together in the real world.
